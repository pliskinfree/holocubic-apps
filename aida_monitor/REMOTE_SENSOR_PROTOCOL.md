# AIDA64 RemoteSensor compatibility notes

This app treats the page generated by AIDA64 as the protocol definition. The
wire format is not a standalone binary LCD protocol: AIDA64 serves an HTML/CSS
document containing its Canvas renderer calls, then streams changing values
through HTML5 Server-Sent Events (SSE) at `/sse`.

## Evidence and compatibility boundary

- AIDA64's own RemoteSensor announcement describes the browser transport as
  HTML5 SSE: <https://forums.aida64.com/topic/2636-remotesensor-lcd-for-smartphones-and-tablets/>
- `RSLCD-Viewer` is an open-source WebView2 shell around the generated URL; it
  delegates layout and drawing to AIDA's page rather than reimplementing the
  renderer: <https://github.com/IAmOrion/RSLCD-Viewer>
- `esp-aida64` extracts selected values from the served page and requires IDs to
  be added manually; it is useful evidence for the HTTP transport, but is not a
  complete layout renderer: <https://github.com/Uddoo/esp-aida64>

No complete open-source, non-browser RemoteSensor renderer was found. The most
reliable compatibility target is therefore the concrete HTML, JavaScript calls,
CSS, and SSE produced by the installed AIDA64 version. Run
`tools/probe_remotesensor.py` after an AIDA64 upgrade to diff that runtime ABI.

## Static layout document

`GET /` returns the whole layout. AIDA uses absolute pixel coordinates and a
fixed preview size (320 x 240 for HoloCubic). The `.rslcd` export is an
XML-like declarative source, while this generated HTML is the canonical runtime
representation because it also contains the exact drawing parameters.

| Generated element | Meaning | Runtime update |
| --- | --- | --- |
| `body` CSS | background color and optional `background-image:url(...)` | reload |
| `div#pageN` | zero-based page container | `PageN` |
| `span#LabelN` | static label | reload |
| `span#SimpleN` | live simple sensor text | `SimpleN` |
| `div#SIN` | SensorItem wrapper | visibility follows `SIVN` |
| `div#SIVN` | SensorItem value | `SIVN` |
| `div#BarNbg > span#BarNfg` | SensorItem bar | `BarNp` |
| `canvas#GphN` | line, area, or histogram graph | `GphNp` |
| `canvas#ArcN` | Arc Gauge | `ArcNp` |
| `img` in an absolute `div` | foreground static/animated image | reload |

CSS is part of the ABI. The parser retains position, size, visibility, colors,
gradients, font family/size/style/weight, decorations, shadows, bar borders and
direction, and explicit `z-index`. Items in one z-plane retain DOM order.

The installed AIDA64 configuration exposes the following `.rslcd`/INI source
fields. They are documented here for diffing, but the app intentionally consumes
their generated CSS/Canvas form so new combinations do not require a second
parallel parser:

| Source group | Observed fields |
| --- | --- |
| document/page | `LCDVER`, `SWVER`, `LCDBGCOLOR`, `LCDPAGE1...N` |
| identity/geometry | `ID`, `LBL`, `ITMX`, `ITMY`, `WID`, `HEI` |
| text | `TXTSIZ`, `FNTNAM`, `TXTCOL`, `TXTBIR`, `SHWLBL`, `SHWVAL`, `SHWUNT`, `UNT` |
| styled SensorItem text | `SHDCOL`, `SHDDIS`, `SHDDEP`, `LBLCOL`, `LBLBIS`, `VALCOL`, `VALBIS`, `UNTCOL`, `UNTBIS`, `UNTWID` |
| SensorItem bar | `SHWBAR`, `BARWID`, `BARHEI`, `BARIND`, `BARPLC`, `BARFS`, `BARFRMCOL`, `BARMIN`, `BARLIM1...3`, `BARMAX`, and each range's `FGC`/`BGC` colors |
| graph | `TYP`, `GPHSTP`, `GPHTCK`, `GRDDNS`, `MINVAL`, `MAXVAL`, `AUTSCL`, `GRDCOL`, `GPHCOL`, `BGCOL`, `FRMCOL`, `GPHBFG`, `SHWSCL`, `SCLCOL`, `SCLBI` |
| arc | `DIA`, `TCK`, `STAANG`, `OPT`, `MIN`, `LIM1...3`, `MAX`, range colors, and value text fields |
| image | `IMGFIL`, `BGIMG`, `RESIZW`, `RESIZH` |

The three-character `*BIS`/`*BIR` flags become generated font weight/style and
text-decoration CSS. Threshold/range colors become the live Arc/Bar colors in
SSE, so the device follows AIDA's own range selection instead of duplicating it.

`body background-image` has special AIDA `BGIMG=1` semantics. It is normalized
to `(0,0,320,240)` and stretched independently on both axes. It is not emitted
as a foreground `<img>` widget.

## Canvas call ABI

The installed AIDA64 page initializes graphs with:

```text
DrawGraph(dst, grapharray, gridoffset, graph_type, step, thickness,
          grid_density, min, max, autoscale, round_to_100,
          show_background, background_color,
          show_frame, frame_color, show_grid, grid_color, graph_color,
          show_scale, font_family, font_color, font_size,
          font_style, font_variant, font_weight, right_align)
```

`graph_type` values observed are `LG` (line), `AG` (area), and `HG`
(histogram). New samples enter at the right edge. Their horizontal distance is
`step + 1`; the scrolling vertical grid offset moves by the same amount.

Arc gauges are initialized with:

```text
DrawArcGauge(dst, thickness, start_angle, percent,
             background_color, active_color,
             fill, fill_color, show_text, text,
             font_family, font_color, font_size,
             font_style, font_variant, font_weight)
```

The generated function bodies remain the final reference for edge cases such as
autoscaling and grid alignment. The probe records their parameter lists and a
hash of the page so changes are visible after an upgrade.

## SSE stream

`GET /sse` uses `text/event-stream`. One `data:` payload may contain multiple
records separated by `{|}`. Fields inside a record are separated by `|`.

```text
Page0
SIV3|42
Bar3p|42|#202020,#151515|#00FF00,#00AA00
Gph4p|42
Arc7p|42|42|#202020|#00FF00
Simple11|CPU Temp 48&deg;C
ReLoad
```

| Command | Fields | Semantics copied from AIDA JavaScript |
| --- | --- | --- |
| `PageN` | none | show page `N`, hide the others |
| `LabelN`, `SimpleN`, `SIVN` | text | set decoded HTML text; empty hides the item |
| `BarNp` | percent, background gradient, foreground gradient | empty percent hides SensorItem; otherwise update bar and show it |
| `GphNp` | numeric sample | newest sample is drawn on the right; empty clears the graph canvas |
| `ArcNp` | percent, text, background color, active color | redraw; empty text hides the gauge |
| `ReLoad` | none | refetch the complete HTML layout after a short delay |

Unknown IDs and commands are ignored so values from a stale/reloading page do
not terminate the app.

### Local gravity paging

HoloCubic maps physical left/right tilt gestures to `key.LEFT` and `key.RIGHT`.
The app changes the already-built RemoteSensor page locally, wraps at both ends,
and handles only the gesture's `START` edge; its matching `SHORT` release is
ignored. A configurable cooldown (1000 ms by default) also absorbs threshold
bounce. After the first local tilt, incoming SSE
values continue updating every page, but SSE `PageN` commands are ignored so
the server cannot immediately pull the display back to `Page0`. The selected
local page is retained and clamped when a `ReLoad` changes the page count.

## HoloCubic compositor

The browser's stacking model is reproduced explicitly:

```text
decoded body background (RGB565, bilinear full-screen stretch)
    -> software Canvas/text/SensorItem layer (RGB565 + A8 coverage)
    -> foreground LVGL image/GIF widgets
```

Two 320 x 240 RGB565 surfaces consume 307,200 bytes total. Image decode buffers
use PSRAM temporarily and are bounded by encoded byte and decoded pixel limits.
Static background alpha is composited before the background surface is cached.

Low-resolution primitives use per-pixel coverage antialiasing for circles,
arcs, and lines. Text retains the TrueType grayscale coverage and optionally
uses a mild RGB or BGR horizontal subpixel filter; the WebUI can disable it for
optical stacks that blur or reorder physical subpixels.

## Known AIDA distinction

SensorPanel-only Custom Gauge assets are not part of the RemoteSensor renderer.
AIDA's supported RemoteSensor gauge primitive is Arc Gauge. This is an upstream
engine distinction, not a missing SSE record that can be recovered from the
RemoteSensor page.
